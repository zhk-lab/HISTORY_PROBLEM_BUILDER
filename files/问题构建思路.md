第一步：
找到合适的网站，进行内容爬取。

Wikipedia Current Events
适合作为“每日重大事件索引”。它按日期整理全球新闻，并附外部来源链接，方便回到某一天找候选事件。缺点是维基本身不应作为最终答案唯一来源，但很适合发现事件。

GDELT Project
适合大规模爬取全球新闻事件。它覆盖多语言新闻，更新频率高，历史数据长，适合批量筛选政治、冲突、外交、灾害等事件。缺点是数据噪声较多，需要后续人工或规则筛选。

IFES ElectionGuide
非常适合政治选举类问题。它提供全球国家级选举、公投、结果、投票率等信息，且结果通常来自官方选举机构。和你的优先领域“政治选举”高度匹配。

Federal Reserve FOMC Calendar + BLS Release Calendar + FRED Release Calendar
适合宏观经济和政策问题，比如美联储利率、CPI、非农、PCE 等。这类问题时间边界清楚、结果明确、官方来源稳定，非常符合手册要求。

Trading Economics Economic Calendar
适合作为宏观事件发现源，覆盖多国经济数据、央行会议、实际值/预测值/前值。建议用它发现事件，再用官方机构网站做最终答案来源。

ReliefWeb
适合公共风险、灾害、人道危机类问题。它由联合国 OCHA 管理，更新频繁，有大量报告和历史页面，适合构建“某灾害是否升级、某机场/地区是否恢复、某援助行动是否发生”等问题。

第二步：
在现有事件爬取结果之上，新增一条独立的“历史预测问题生成流水线”：读取 `data/event/events_*.jsonl`，先用规则筛出适合构造问题的事件，再调用大模型 agent 生成候选问题，然后由验证层打风险标签，最后输出给人工复核的 CSV/JSONL。
